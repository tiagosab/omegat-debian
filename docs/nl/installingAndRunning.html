Linux (Intel)-gebruikers

Welk pakket downloaden ?

Heeft u een Java-implementatie die compatibel is met SUN's Java 1.5 JRE?

Ja: 

download OmegaT_2.0.x_Without_JRE.zip. Dit pakket kan worden gebruikt op elk platform waar een Java 1.5 JRE compatibele JRE is geïnstalleerd.

Nee / Ik weet het niet:

download OmegaT_2.0.x_Linux.tar.gz. Dit pakket is gebundeld met SUN's Java Runtime Environment. Deze JRE zal geen problemen veroorzaken met andere Java-implementaties die misschien al op uw systeem geïnstalleerd zijn.

OmegaT-installatie

Pak eenvoudigweg het gedownloade bestand uit om OmegaT te installeren. Dit zal een map omegat/ maken in de werkmap waarin u alle nodige bestanden vindt om OmegaT uit te voeren. Uitpakken van het .tar.gz bestand:

$ tar xf downloaded_file.tar.gz

OmegaT toevoegen aan uw menu's (KDE) of panelen (Gnome)

KDE-gebruikers

U kunt OmegaT als volgt aan uw menu's toevoegen:

  • Controlecentrum - Bureaublad - Panelen - Menu's - Bewerken K Menu - Bestand - Nieuw item/Nieuw submenu

  • Dan, na het selecteren van een passend menu, voeg een submenu/item toe met Bestand - Nieuw submenu en Bestand - Nieuw item. Voer OmegaT in als de naam van het nieuwe item.

  • Gebruik, in het veld "Commando", de navigatieknop om uw OmegaT startscript te vinden (het bestand genaamd OmegaT in de uitgepakte map) en selecteer dat.

  • Klik op de pictogrammenknop (rechts van de velden Naam/Beschrijving/Commentaar) - Andere pictogrammen - Bladeren en navigeer naar de submap /images in de OmegaT-toepassingsmap. Selecteer het pictogram OmegaT.png.

  • Sla tenslotte de wijzigingen op met Bestand - Opslaan.

Gnome-gebruikers

U kunt OmegaT als volgt toevoegen aan uw paneel (de balk boven in het scherm):

  • Rechtsklik op het paneel - Nieuwe starter toevoegen.

  • Voer "OmegaT" in in het veld "Naam"; gebruik, in het veld "Commando", de navigatieknop om uw OmegaT-startscript te vinden (het bestand genaamd OmegaT in de uitgepakte map). Selecteer dat en bevestig met OK.

  • Klik op de pictogramknop, klik dan op Bladeren... en navigeer naar de submap /images in de OmegaT-toepassingsmap. Selecteer het pictogram OmegaT.png. GNOME zou mogelijk de pictogrambestanden niet in de beschikbare indelingen kunnen tonen en zou zijn ingesteld om een SVG-bestand te verwachten, maar als de map is geselecteerd, zouden de bestanden toch verschijnen en kan OmegaT.png worden geselecteerd.

OmegaT uitvoeren

Als OmegaT eenmaal is geïnstalleerd, kunt u het direct vanaf de opdrachtregel starten, u kunt een script maken dat de startparameters voor de opdrachtregel bevat of u kunt op OmegaT.jar klikken om het direct uit te voeren. Methoden verschillen, afhankelijk van de distributie. Zorg ervoor dat de instellingen van uw PATH juist zijn en dat .jar-bestanden juist zijn gekoppeld aan een Java-starter.

download OmegaT_2.0.x_Windows_without_JRE.exe. Dit pakket kan worden gebruikt op elk Windows-platform waar een Java 1.5 JRE compatibele JRE is geïnstalleerd.

Nee / Ik weet het niet:

download OmegaT_2.0.x_Windows.exe. Dit pakket is gebundeld met SUN's Java Runtime Environment. Deze JRE zal geen problemen veroorzaken met andere Java-implementaties die misschien al op uw systeem geïnstalleerd zijn.

OmegaT-installatie

Dubbelklik op het programma dat u hebt gedownload en volg de instructies om OmegaT te installeren.

OmegaT toevoegen aan het startmenu of bureaublad

Gedurende de installatie zal het programma u vragen of u een map in het menu Start wilt maken en of u een snelkoppeling op het bureaublad en het systeemvak wilt maken.

U kunt daarna nog steeds een snelkoppeling maken door OmegaT.exe naar het bureaublad of menu Start te slepen om het van daaruit te koppelen.

OmegaT uitvoeren

Als OmegaT eenmaal is geïnstalleerd, kunt u klikken op OmegaT.jar om het direct te starten, of u kunt het direct vanaf de opdrachtregel starten.

De eenvoudigste manier is echter om het programma OmegaT.exe uit te voeren. De startparameters zullen in dit geval worden gelezen uit het bestand OmegaT.l4J.ini, dat zich in dezelfde map bevindt als het exe-bestand en dat u kunt bewerken om uw instellingen weer te geven. Het volgende INI-bestand zal 1GB aan geheugen reserveren, Frans als gebruikerstaal instellen en Portugal als het land:

# OmegaT.exe runtime configuratie
#
# verwijder de '#' voor de '-' om een parameter te gebruiken
# Geheugen
-Xmx1024M
#taal gebruikersinterface
-Duser.language=FR
# Land instellen
-Duser.country=PT
# Anti-aliasing (alleen Java 1.4 en 1.5 )
#-Dswing.aatext=true
# Entiteit uitbreiding voor XML (alleen Java 1.4 )
#-DentityExpansionLimit=400000

MacOSX-gebruikers

Welk pakket downloaden ?

OmegaT 2.0 vereist Java 1.5 om te kunnen worden uitgevoerd. Mac OSX 10.4 (Tiger)-gebruikers zouden zich er van moeten overtuigen dat zij de laatste versie van Java gebruiken voordat OmegaT wordt geïnstalleerd. Gebruik "Software Update..." uit het menu Apple om uw versie van Java bij te werken.

Download OmegaT_2.0.x_Mac.zip.

OmegaT-installatie

Dubbelklik op OmegaT_2.0.x_Mac.zip om het uit te pakken. Dit zal de de map OmegaT maken, met het OmegaT-pictogram er op. De map bevat 2 bestanden: documentation.html en OmegaT.app. Kopieer de map naar een toepasselijke map (bijv. Toepassingen). Als u dit gedaan heeft kunt u het bestand OmegaT_2.0.X_Mac.zip verwijderen.

OmegaT toevoegen aan het dock

Voeg OmegaT toe aan het dock door het bestand OmegaT.app te slepen en neer te zetten op het dock.

OmegaT uitvoeren

Dubbelklik op OmegaT.app of klik op zijn locatie in het dock

Indien u de startparameters voor Java wilt wijzigen kunt u de standaardinstellingen voor OmegaT.app aanpassen door het bestand Info.plist opgeslagen in OmegaT.app/Contents/ te bewerken. Open het bestand "Info.plist" door er met rechts op te klikken en het te openen met de tekstbewerker van uw keuze. U kunt zowel het gedeelte Properties als het gedeelte VMOptions aanpassen. Indien u waarden wilt toevoegen aan VMOptions plaatst u er een spatie tussen. Bijvoorbeeld: -Xmx1024M -Duser.language=ja zal OmegaT starten met 1024MB geheugen en met de gebruikersinterface in het Japans.

Direct starten van OmegaT.app zal u niet in staat stellen om meerdere instanties van OmegaT.app uit te voeren. Indien u meerdere instanties van OmegaT.app moet uitvoeren, gebruik dan in plaats daarvan het bestand JavaApplicationStub dat is opgeslagen in OmegaT.app/ Contents/MacOS/.

Het is niet mogelijk om de Lucene stemmer te gebruiken uit het OmegaT.app-pakket. U moet OmegaT uitvoeren vanaf de opdrachtregel (zie hieronder).

Roep het bestand OmegaT.jar aan dat is opgeslagen in OmegaT.app/Contents/Resources/ Java/ om de startparameters voor Java vanaf de opdrachtregel te wijzigen. Opstarten van OmegaT.app vanaf de opdrachtregel is equivalent aan het gebruiken van het "algemene" pakket van OmegaT. U zult in staat zijn om meerdere instanties van OmegaT uit te voeren, en ook om de Lucene stemmer te gebruiken. Kijk bij "Opstarten vanaf de opdrachtregel" hieronder voor meer informatie.

Klik met rechts op OmegaT.app en selecteer "Toon inhoud pakket", of "cd" er direct vanaf de opdrachtregel naartoe om toegang te krijgen tot elke locatie binnen het OmegaT.app-pakket.

Mac OSX-goodies

OmegaT.app kan worden benaderd vanuit de Services die Mac OSX aanbiedt. U kunt dus overal in OmegaT een woord selecteren en de Services gebruiken om dat woord te controleren in de door de Services aangeboden toepassingen voor woordenboeken, in Spotlight of in Google. U kunt ook AppleScript of Automator gebruiken om Services of scripts te maken die veel voorkomende acties kan automatiseren.

Starten vanaf de opdrachtregel

Normaal gesproken is het niet nodig om OmegaT te starten vanaf de opdrachtregel. De methode om OmegaT te starten vanaf de opdrachtregel maakt het echter mogelijk om bepaalde aspecten van het gedrag van het programma te beheren door middel van aanpassingen. Er zijn twee manieren om OmegaT te starten met behulp van de opdrachtregel.

Methode 1: openen van een venster voor de opdrachtregel

Een venster voor de opdrachtregel wordt ook wel een "terminalvenster" genoemd. Op Windows wordt het een "MS-DOSvenster" genoemd en is het beschikbaar via het Startmenu, in Alle programma's → Bureau-accessoires, via het item "Opdrachtprompt". Het equivalent op Mac OSX is de toepassing Terminal die is geplaatst in Programma's → Hulpprogramma's.

U moet normaal gesproken twee opdrachten typen om OmegaT te starten. De eerste hiervan is:

cd {map}

waar {map} de naam is van de map, met het complete pad, waarin uw OmegaT-programma - specifiek het bestand OmegaT.jar - is geplaatst. In de praktijk zal deze opdracht er ongeveer zo uitzien:

Op Windows
cd C:\Program Files\OmegaT
Op Mac OSX
cd <locatie van OmegaT.app>/OmegaT.app/Contents/Resources/Java/
Op Linux
cd /usr/local/omegat

Deze opdracht wijzigt de map naar de map die het uitvoerbare OmegaT-bestand bevat. De tweede opdracht is de opdracht die OmegaT echt start. In zijn meest basale vorm is deze opdracht:

java -jar OmegaT.jar

Let op de hoofdletters - die zijn belangrijk.

Deze methode heeft het bijzondere voordeel dat het geschikt is voor debuggen: als een fout optreedt gedurende het gebruiken van het programma wordt een foutbericht weergegeven in het terminalvenster dat bruikbare informatie kan bevatten over de oorzaak.

Methode 2: wijzigen van het bestand/startscript .bat

Methode 1 is een enigszins onpraktische manier van het routinematig starten van een programma. Om deze reden zijn de twee hierboven beschreven opdrachten opgenomen in een bestand (een "script", ook wel een ".bat-bestand" genoemd op Windows-systemen).

Als dit bestand wordt uitgevoerd worden de daarin opgenomen opdrachten automatisch uitgevoerd. Derhalve is het voldoende om dit bestand aan te passen om wijzigingen aan te brengen in de startopdracht. Als OmegaT wordt gestart vanuit het bestand worden dan de nieuwe opdrachten uitgevoerd.

Argumenten voor de startopdracht

De basisopdracht is hierboven al genoemd. Wijzigingen voor deze opdracht behelzen het daaraan toevoegen van "argumenten". Argumenten worden toegevoegd ná het initiële "java" en vóór het "-jar OmegaT.jar" gedeelte. Onthoud dat u in Windows het bestand OmegaT.l4J.ini kunt wijzigen om uw voorkeuren weer te geven. Op Mac OSX kunt u Info.plist wijzigen dat is geplaatst in OmegaT.app/Contents/ om hetzelfde te doe.

Een lijst van mogelijke argumenten wordt hieronder weergegeven. U kunt meer informatie over de argumenten krijgen door man java in het terminalvenster te typen - man is een opdracht voor het terminalvenster dat een handleiding van de geselecteerde opdracht weergeeft. Uw systeem kan wel of niet een handleiding voor de opdracht java in uw taal hebben.

Taal voor de gebruikersinterface: -Duser.language=XX

     

Normaal gesproken, dat is als OmegaT wordt gestart zonder enig argument, detecteert het programma eerst de taal van het besturingssysteem van de gebruiker. Als een gebruikersinterface in deze taal beschikbaar is gebruikt OmegaT die. Dus, als het besturingssysteem van de gebruiker Russisch is en OmegaT is vertaald in het Russisch, wordt OmegaT weergegeven met een Russische gebruikersinterface, Russische menu's, etc. Als de taal van het besturingssysteem van de gebruiker niet beschikbaar is gebruikt OmegaT standaard Engels. Dit is het standaardgedrag.

Het argument "-Duser.language=XX" zorgt ervoor dat OmegaT de gespecificeerde taal gebruikt, in plaats van de taal van het besturingssysteem van de gebruiker. "XX" in de opdracht staat voor de twee-lettercode van de gewenste taal. De opdracht, om OmegaT te starten met een Franse interface (bijvoorbeeld op een Russisch besturingssysteem), zou daarom zijn:

java -Duser.language=fr -jar OmegaT.jar

Instellen land gebruiker: -Duser.country=XX

Naast de taal kunt u ook het land specificeren, bijvoorbeeld CN of TW in geval van de Chinese taal. U moet zowel de taal als het land specificeren om de juiste gids Snel starten te krijgen. Dit is zelfs nodig als er slechts één combinatie beschikbaar is, zoals pt_BR in geval van Portugees / Braziliaans.

     

Antialiasing lettertype: -Dswing.aatext=true

     

Dit argument beïnvloedt de anti-aliasing van de lettertypen, waarbij hun weergave wordt verbeterd.

Anti-aliasing lettertype (Java 6 en hoger) : -Dawt.useSystemAAFontSettings=XX

     

 Mogelijke waarden voor "XX" zijn "false", "on", "gasp" en "lcd". Linux-gebruikers zouden kunnen vinden dat, afhankelijk van het geselecteerde lettertype en grootte, "on" of "lcd" het uiterlijk van de lettertypen verbeteren.

Toewijzen geheugen: -XmxZZM

     

Deze opdracht wijst meer geheugen toe aan OmegaT. Standaard wordt 256 Mb toegewezen, dus er is geen voordeel te behalen door minder dan dit getal toe te wijzen. "ZZ" staat voor de toegewezen geheugenruimte in megabytes. De opdracht om OmegaT te starten met toewijzing van 1024 Mb (1 gigabyte) aan geheugen luidt dan ook:

java -Xmx1024M -jar OmegaT.jar

Argumenten mogen worden gecombineerd: Voor het starten van OmegaT met alle hierboven genoemde voorbeelden zou de opdracht zijn:
java -Dswing.aatext=true -Duser.language=pt -Duser.country=BR -Xmx1024M -jar OmegaT.jar

OmegaT bouwen vanuit de bron

Welk pakket downloaden ?

Download OmegaT_2.0.x_Source.zip
Dit pakket bevat de bronbestanden die overeenkomen met de vrijgegeven versie en kan worden gebruikt op elk systeem waarop een Java 1.5 JDK of compatibel Java-systeem is geïnstalleerd. U zult ook de ant-utility nodig hebben om OmegaT te bouwen.
OmegaT gebruikt SubVersion voor het beheren van versies. De meest recente code is beschikbaar op de SVN-server:
https://omegat.svn.sourceforge.net/svnroot/omegat/trunk

OmegaT bouwen

Pak het bestand OmegaT_2.0.x_Source.zip uit en geef de map OmegaT_2.0.x_Source op of voer de map ./omegat/ in voor de checkout-code van SVN. Controleer of een bestand build.xml in die map aanwezig is. Typ dan op de opdrachtregel:

$ ant jar release

Dit zal een volledige distributie van OmegaT maken in de map ./dist/, waarin u alle benodigde bestanden zult vinden om OmegaT uit te voeren.

Java Web Start gebruiken

Door Java Web Start-technologie (onderdeel van Java 1.5 en hoger) te gebruiken, kunnen zelfstandige Java softwaretoepassingen worden uitgevoerd met één enkele klik over het netwerk. Java Web Start zorgt er voor dat de meest recente versie van de toepassing al worden uitgevoerd, net als de juiste versie van de Java Runtime Environment (JRE). Laad de volgende URL in uw browser om OmegaT voor de eerste keer met Java Web Start te starten:
    http://omegat.sourceforge.net/webstart/OmegaT.jnlp

Download het bestand OmegaT.jnlp en klik er dan op. Gedurende de installatie kunt u, afhankelijk van uw besturingssysteem, een aantal veiligheidswaarschuwingen ontvangen. Het certificaat is zelf-ondertekend door "Didier Briel". De toestemmingen die u deze versie geeft (welke kunnen worden vermeld als "onbeperkte toegang tot de computer") zijn identiek aan de toestemmingen die u aan de lokale versie geeft, zij geven toegang tot de harde schijf van de computer. Opeenvolgend klikken op OmegaT.jnlp zal controleren op upgrades, indien u online bent, ze installeren als er zijn, en dan OmegaT starten. Na de initiële installatie kunt u natuurlijk OmegaT.jnlp ook gebruiken als u offline bent.

Privacy: OmegaT Java Web Start slaat geen informatie van u op buiten de computer waarop u het uitvoert. De toepassing wordt alleen op uw machine uitgevoerd. Uw documenten en vertaalgeheugens blijven op uw computer, en het OmegaT project zal geen toegang hebben tot uw werk of informatie.

Merk op dat als u argumenten voor opstartopdrachten nodig hebt (zie boven), dat u de normale installatie moet gebruiken.
