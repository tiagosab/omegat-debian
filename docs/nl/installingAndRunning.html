Linux (Intel)-gebruikers

Welk pakket downloaden ?

Heeft u een Java-implementatie die compatibel is met SUN's Java 1.5 JRE?

Ja: 

Download OmegaT_2.0.x_Without_JRE.zip. Dit pakket kan worden gebruikt op elk platform waar een Java 1.5 JRE compatibele JRE is geïnstalleerd.

Nee / Ik weet het niet:

download OmegaT_2.0.x_Linux.tar.gz. Dit pakket is gebundeld met SUN's Java Runtime Environment. Deze JRE zal geen problemen veroorzaken met andere Java-implementaties die misschien al op uw systeem geïnstalleerd zijn.

OmegaT-installatie

Pak eenvoudigweg het gedownloade bestand uit om OmegaT te installeren. Dit zal een map omegat/ maken in de werkmap waarin u alle nodige bestanden vindt om OmegaT uit te voeren. Uitpakken van het .tar.gz bestand:

$ tar xf downloaded_file.tar.gz

OmegaT toevoegen aan uw menu's (KDE) of panelen (Gnome)

KDE-gebruikers

U kunt OmegaT als volgt aan uw menu's toevoegen:

  • Controlecentrum - Bureaublad - Panelen - Menu's - Bewerken K Menu - Bestand - Nieuw item/Nieuw submenu

  • Dan, na het selecteren van een passend menu, voeg een submenu/item toe met Bestand - Nieuw submenu en Bestand - Nieuw item. Voer OmegaT in als de naam van het nieuwe item.

  • Gebruik, in het veld "Commando", de navigatieknop om uw OmegaT startscript te vinden (het bestand genaamd OmegaT in de uitgepakte map) en selecteer dat.

  • Klik op de pictogrammenknop (rechts van de velden Naam/Beschrijving/Commentaar) - Andere pictogrammen - Bladeren en navigeer naar de submap /images in de OmegaT-toepassingsmap. Selecteer het pictogram OmegaT.

  • Sla tenslotte de wijzigingen op met Bestand - Opslaan.

Gnome-gebruikers

U kunt OmegaT als volgt toevoegen aan uw paneel (de balk boven in het scherm):

  • Rechtsklik op het paneel - Nieuwe starter toevoegen.

  • Voer "OmegaT" in in het veld "Naam"; gebruik, in het veld "Commando", de navigatieknop om uw OmegaT-startscript te vinden (het bestand genaamd OmegaT in de uitgepakte map). Selecteer dat en bevestig met OK.

  • Klik op de pictogramknop, en druk dan op Bladeren... en navigeer naar de submap /images in de toepassingenmap OmegaT. Selecteer het pictogram OmegaT_icon.png. GNOME kan weigeren om de pictogrambestanden weer te geven in de beschikbare indelingen en in het begin een SVGbestand verwachten, maar als de map wordt geselecteerd, zouden de bestanden moeten verschijnen en kan OmegaT_icon.png worden geselecteerd.

OmegaT uitvoeren

Als OmegaT eenmaal is geïnstalleerd, kunt u het direct vanaf de opdrachtregel starten, u kunt een script maken dat de startparameters voor de opdrachtregel bevat of u kunt op OmegaT.jar klikken om het direct uit te voeren. Methoden verschillen, afhankelijk van de distributie. Zorg ervoor dat de instellingen van uw PATH juist zijn en dat .jar-bestanden juist zijn gekoppeld aan een Java-starter.

download OmegaT_2.0.x_Windows_without_JRE.exe. Dit pakket kan worden gebruikt op elk Windows-platform waar een Java 1.5 JRE compatibele JRE is geïnstalleerd.

Nee / Ik weet het niet:

download OmegaT_2.0.x_Windows.exe. Dit pakket is gebundeld met SUN's Java Runtime Environment. Deze JRE zal geen problemen veroorzaken met andere Java-implementaties die misschien al op uw systeem geïnstalleerd zijn.

OmegaT-installatie

Dubbelklik op het programma dat u hebt gedownload en volg de instructies om OmegaT te installeren.

OmegaT toevoegen aan het menu Start of bureaublad

Gedurende de installatie zal het programma u vragen of u een map in het menu Start wilt maken en of u een snelkoppeling op het bureaublad en het systeemvak wilt maken.

U kunt daarna nog steeds een snelkoppeling maken door OmegaT.exe naar het bureaublad of menu Start te slepen om het van daaruit te koppelen.

OmegaT uitvoeren

Als OmegaT eenmaal is geïnstalleerd, kunt u klikken op OmegaT.jar om het direct te starten, of u kunt het direct vanaf de opdrachtregel starten.

De eenvoudigste manier is echter om het programma OmegaT.exe uit te voeren. De startparameters zullen in dit geval worden gelezen uit het bestand OmegaT.l4J.ini, dat zich in dezelfde map bevindt als het exe-bestand en dat u kunt bewerken om uw instellingen weer te geven. Het volgende INI-bestand zal 1GB aan geheugen reserveren, Frans als gebruikerstaal instellen en Portugal als het land:

# OmegaT.exe runtime configuratie
#
# Verwijder de '#' voor de '-' om een parameter te gebruiken
# Geheugen
-Xmx1024M
# Taal gebruikersinterface
-Duser.language=FR
# Land instellen
-Duser.country=PT
# Anti-aliasing (alleen Java 1.5 )
#-Dswing.aatext=true
# Instellingen om op internet te gaan van achter een proxy
#-Dhttp.proxyHost=192.168.1.1
#-Dhttp.proxyPort=3128

Mac OS X-gebruikers

Welk pakket downloaden ?

OmegaT 2.0 vereist Java 1.5 om te kunnen worden uitgevoerd. Gebruikers van Mac OS X 10.4 (Tiger) zouden er voor moeten zorgen dat zij de laatste versie van java gebruiken voordat zij OmegaT installeren. Gebruik "Software Update..." uit het menu Apple om uw versie van java bij te werken.

Download OmegaT_2.0.x_Mac.zip.

OmegaT-installatie

Dubbelklik op OmegaT_2.0.x_Mac.zip om het uit te pakken. Dit zal een map maken die OmegaT heet, met het pictogram van OmegaT er op. De map bevat 2 bestanden: documentation.html en OmegaT.app. Kopieer de map naar een toepasselijke map (bijv. Toepassingen). Als u dit gedaan heeft kunt u het bestand OmegaT_2.0.x_Mac.zip verwijderen.

OmegaT toevoegen aan het dock

Voeg OmegaT toe aan het dock door het bestand OmegaT.app te slepen en neer te zetten op het dock.

OmegaT uitvoeren

Dubbelklik op OmegaT.app of klik op zijn locatie in het dock

Als u de Java-startparameters vanaf de opdrachtregel wilt wijzigen kunt u de standaardinstellingen van OmegaT.app aanpassen door het bewerken van het bestand Info.plist dat zich bevind in OmegaT.app/Contents/. Open het bestand "Info.plist" door er met rechts op te klikken en de tekstbewerker van uw keuze te kiezen. U kunt het gedeelte Properties en ook het gedeelte VMOptions aanpassen. Indien u waarden wilt toevoegen aan VMOptions, plaatst u er een spatie tussen. Bijvoorbeeld: -Xmx1024M -Duser.language=ja zal OmegaT opstarten met 1024MB aan geheugen en de gebruikersinterface in het Japans.

Direct opstarten van OmegaT.app maakt het niet mogelijk om meerdere instanties van OmegaT.app uit te voeren. Als u meerdere instanties van OmegaT.app moet uitvoeren, gebruik dan in plaats daarvan het bestand JavaApplicationStub dat zich bevind in OmegaT.app/ Contents/MacOS/.

Het is niet mogelijk om de Lucene stemmer te gebruiken uit het pakket OmegaT.app. U moet OmegaT uitvoeren vanaf de opdrachtregel (zie hieronder).

Roep het bestand OmegaT.jar aan dat zich bevind in OmegaT.app/Contents/Resources/ Java/ om startparameters van Java te wijzigen vanaf de opdrachtregel. Opstarten van OmegaT.app vanaf de opdrachtregel is hetzelfde als het gebruiken van het "algemene" pakket van OmegaT. U zult in staat zijn om meerdere instanties van OmegaT uit te voeren en ook om de Lucene stemmer te gebruiken. Bekijk "Starten vanaf de opdrachtregel" hieronder voor meer informatie.

Klik met rechts op OmegaT.app en selecteer "Show Package Contents", of "cd" daar direct naar toe vanaf de opdrachtregel om toegang te krijgen tot een locatie binnen het pakket OmegaT.app.

OmegaT.app kan worden bereikt vanuit de Services die Mac OS X bied. U kunt dus ergens in OmegaT een woord selecteren en de Services gebruiken om het te controleren in de door uw Services verschafte woordenboektoepassingen, in Spotlight of in Google. U kunt ook AppleScript of Automator gebruiken om Services of scripts te maken die acties die veel voorkomen automatiseren.

Andere systemen

Welk pakket downloaden ?

OmegaT is beschikbaar, gebundeld met een SUN Java JRE voor Linux (Intel x86)- en Windows-platformen. Gebruikers van andere platformen (Linux PowerPC, Linux x64, Solaris SPARC/x86/x64, Windows x64 etc) moeten een werkende compatibele Java JRE op hun systeem hebben om OmegaT te kunnen gebruiken.

Heeft u een Java-implementatie die compatibel is met SUN's Java 1.5 JRE?

Ja:

Download OmegaT_2.0.x_Without_JRE.zip. Dit pakket kan worden gebruikt op elk platform waar een Java 1.5 JRE compatibele JRE is geïnstalleerd.

Ik weet het niet:

Open een terminal en type "java -version". Als een "command not found" of soortgelijk bericht wordt teruggegeven, is het waarschijnlijk dat Java niet is geïnstalleerd op uw systeem.

Nee:
Zorg voor een Java JRE voor uw systeem (zie hieronder) en download OmegaT_2.0.x_Without_JRE.zip. SUN levert JRE's voor Solaris SPARC/x86 (Java 1.5) en voor Linux x64, Solaris x64, Windows x64 (Java 1.5): http://java.sun.com/products/archive/. IBM levert JRE's voor Linux PowerPC:
http://www.ibm.com/developerworks/java/jdk/linux/download.html
Volg de installatie-instructies van het pakket dat u nodig hebt.

OmegaT-installatie

Pak eenvoudigweg het bestand OmegaT_2.0.x_Without_JRE.zip uit om OmegaT te installeren. Dit maakt een map ./OmegaT_2.0.x_Without_JRE/ in de werkmap waarin u alle benodigde bestanden zult vinden om OmegaT uit te voeren.

Starten vanaf de opdrachtregel

Normaal gesproken is het niet nodig om OmegaT te starten vanaf de opdrachtregel. De methode om OmegaT te starten vanaf de opdrachtregel maakt het echter mogelijk om bepaalde aspecten van het gedrag van het programma te beheren door middel van aanpassingen. Er zijn twee manieren om OmegaT te starten met behulp van de opdrachtregel.

Methode 1: openen van een venster voor de opdrachtregel

Een venster voor de opdrachtregel wordt ook wel een "terminalvenster" genoemd. Op Windows wordt het een "MS-DOSvenster" genoemd en is het beschikbaar via het menu Start, in Alle programma's → Bureau-accessoires, via het item "Opdrachtprompt". Het equivalent voor de Mac OS X is de toepassing Terminal die zich bevind in Programma's → Hulpprogramma's.

U moet normaal gesproken twee opdrachten typen om OmegaT te starten. De eerste hiervan is:

cd {map}

waar {map} de naam is van de map, met het complete pad, waarin uw OmegaT-programma - specifiek het bestand OmegaT.jar - is geplaatst. In de praktijk zal deze opdracht er ongeveer zo uitzien:

Op Windows
cd C:\Program Files\OmegaT
Op Mac OS X
cd <OmegaT.app location>/OmegaT.app/Contents/Resources/Java/
Op Linux
cd /usr/local/omegat

Deze opdracht wijzigt de map naar de map die het uitvoerbare OmegaT-bestand bevat. De tweede opdracht is de opdracht die OmegaT echt start. In zijn meest basale vorm is deze opdracht:

java -jar OmegaT.jar

Let op de hoofdletters - die zijn belangrijk.

Deze methode heeft het bijzondere voordeel dat het geschikt is voor debuggen: als een fout optreedt gedurende het gebruiken van het programma wordt een foutbericht weergegeven in het terminalvenster dat bruikbare informatie kan bevatten over de oorzaak.

Methode 2: wijzigen van het bestand/startscript .bat

Methode 1 is een enigszins onpraktische manier van het routinematig starten van een programma. Om deze reden zijn de twee hierboven beschreven opdrachten opgenomen in een bestand (een "script", ook wel een ".bat-bestand" genoemd op Windows-systemen).

Als dit bestand wordt uitgevoerd worden de daarin opgenomen opdrachten automatisch uitgevoerd. Derhalve is het voldoende om dit bestand aan te passen om wijzigingen aan te brengen in de startopdracht. Als OmegaT wordt gestart vanuit het bestand worden dan de nieuwe opdrachten uitgevoerd.

Argumenten voor de startopdracht

De basisopdracht is hierboven al genoemd. Wijzigingen voor deze opdracht behelzen het daaraan toevoegen van "argumenten". Deze worden in detail beschreven in Argumenten voor de opdrachtregel.

OmegaT bouwen vanuit de bron

Welk pakket downloaden ?

Download OmegaT_2.0.x_Source.zip
Dit pakket bevat de bronbestanden die overeenkomen met de vrijgegeven versie en kan worden gebruikt op elk systeem waarop een Java 1.5 JDK of compatibel Java-systeem is geïnstalleerd. U zult ook de ant-utility nodig hebben om OmegaT te bouwen.
OmegaT gebruikt SubVersion voor het beheren van versies. De meest recente code is beschikbaar op de SVN-server:
https://omegat.svn.sourceforge.net/svnroot/omegat/trunk

OmegaT bouwen

Pak het bestand OmegaT_2.0.x_Source.zip uit en geef de map OmegaT_2.0.x_Source op of voer de map ./omegat/ in voor de checkout-code van SVN. Controleer of een bestand build.xml in die map aanwezig is. Typ dan op de opdrachtregel:

$ ant jar release

Dit zal een volledige distributie van OmegaT maken in de map ./dist/, waarin u alle benodigde bestanden zult vinden om OmegaT uit te voeren.

Java Web Start gebruiken

Het gebruiken van Java Web Start technologie (onderdeel van Java 1.5 en hoger), zelfstandige Java softwaretoepassingen kan worden uitgevoerd met één enkele klik over het netwerk. Java Web Start zorgt er voor dat de meest recente versie van de toepassing zal worden uitgevoerd, evenals de juiste versie van de Java Runtime Environment (JRE). Laad de volgende URL in uw browser om OmegaT voor de eerste keer te starten met Java Web Start:
    http://omegat.sourceforge.net/webstart/OmegaT.jnlp

Download het bestand OmegaT.jnlp en klik er op. Gedurende de installatie, afhankelijk van uw besturingssysteem, zou u verschillende beveiligingswaarschuwingen kunnen ontvangen. Het certificaat is zelf-ondertekend door "Didier Briel". De rechten die u aan deze versie geeft (welke kunnen worden omschreven als een "onbeperkte toegang tot de computer") zijn identiek aan de rechten die u geeft voor de locale versie, d.i., zij geven toegang tot de harde schijf van de computer. Opvolgende klikken op OmegaT.jnlp zal leiden tot het controleren op upgrades, ze installeren, als ze er zijn, en dan OmegaT starten. Na de initiële installatie kunt u natuurlijk OmegaT.jnlp ook gebruiken als u off line bent.

Privacy: OmegaT Java Web Start slaat geen informatie van u op buiten de computer waarop het wordt uitgevoerd. De toepassing wordt slechts uitgevoerd op uw machine. Uw documenten en vertaalgeheugens blijven op uw computer, en het OmegaTproject zal geen toegang hebben tot uw werk of informatie.

Merk op dat als u argumenten voor het opstarten wilt gebruiken (zie boven), u de normale installatie zal moeten gebruiken.
