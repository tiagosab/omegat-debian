Instalación y configuración

Instalación

La instalación consta de tres pasos. Uno, instalar el J2RE (si no lo está ya). Dos, instalar OmegaT. Tres, configurar (sólo es necesario en Linux). La instalación del J2RE y de OmegaT puede realizarse en cualquier orden. La configuración no es esencial, pero permite ejecutar la aplicación con mayor facilidad.

El entorno de ejecución Java (Java Runtime Environment)

El entorno de ejecución java (Java Runtime Environment, J2RE) le permite ejecutar en su sistema cualquier programa escrito en Java. Está disponible tanto como producto independiente como formando parte del kit de desarrollo de Java.

Java está asimismo disponible como complemento para los navegadores de Internet. Esta versión de Java no le permite ejecutar OmegaT.

JavaScript es un producto totalmente diferente y técnicamente no está en modo alguno relacionado con Java.

Si desea utilizar OmegaT, el J2RE debe estar instalado en su sistema. OmegaT no funcionará sin él. No intente que OmegaT funcione sin el J2RE. ¡Estará perdiendo el tiempo!

Si no está seguro de que el J2RE esté instalado en su sistema, pruebe simplemente a instalar OmegaT para ver si funciona (consulte más abajo la instalación de OmegaT). Si no ocurre nada, o si recibe mensajes de error, es casi seguro que el entorno de ejecución Java no está instalado. En este caso, consígalo e instálelo.

(Nota: a fin de confundir a sus clientes, Sun cambia con regularidad el nombre de sus productos. Así, todas las versiones del JRE posteriores a la 1 se conocen como "Java 2", de ahí que el Java Runtime Environment sea ahora el J2RE. El Java Runtime Environment puede ser asimismo conocido como Java Virtual Machine, o JVM, y en Macintosh, puede conocerse como MRJ. El kit de desarrollo ha sido conocido como Java Developer's Kit (JDK), Software Developer's Kit (SDK), y ahora también como J2SE, J2EE y J2ME, según la versión. Al menos, eso es lo que creo. Pero soy uno de los confundidos).

Instalación del entorno de ejecución Java

La única plataforma que no requiere la instalación del entorno de ejecución Java es MacOSX. En MacOSX, el JRE viene preinstalado y preconfigurado. Este es asimismo el caso en algunas distribuciones de Linux.

Usuarios de Windows: pueden comprobar qué versión del entorno de ejecución Java está instalada (caso de estarlo) con Inicio > Configuración > Panel de control. Si el J2RE está ya instalado, verán un panel de control denominado "Java(TM)2 Runtime Environment, Standard Edition 1.3.0_01" o similar.

Usuarios de Linux: tecleen simplemente java -version en la línea de mandatos. Se obtendrá la versión instalada (suponiendo por supuesto que esté instalada en el sistema).

Se puede descargar el J2RE de los enlaces que se encuentran más abajo. Es gratuito (observe las condiciones de la licencia), pero las convenciones de denominación pueden hacer que el encontrar la versión adecuada resulte un tanto confuso. Necesitará la versión indicada para su plataforma (Linux, Windows, etc.). El tamaño del J2RE es de unos 14 MB.

OmegaT 1.4.4 debería funcionar con cualquier versión del J2RE a partir de la 1.2. Sin embargo, recomendamos obtener la última versión del J2RE, ya que las versiones más antiguas de Java pueden no soportar ciertas características.

Como alternativa al J2RE, puede instalarse el kit de desarrollo. El kit de desarrollo contiene su propia copia del J2RE, y también permite modificar OmegaT.

Los usuarios de Windows y Linux pueden descargar el Java Runtime Environment de:

Sun (todas las plataformas)

Blackdown (Linux)

Se sabe que las versiones anteriores de OmegaT pueden ejecutarse en el MRJ para Macintosh OS 9. La versión 1.4.4 puede que funcione pero no se ha probado en esta plataforma. Puede descargarse el MRJ del sitio en Internet de Apple.

Instale el J2RE siguiendo las instrucciones que lo acompañan.

Instalación de OmegaT

Todas las plataformas

Extraiga simplemente los contenidos del archivo en el directorio de su elección. El archivo puede estar en formato *.zip o *.tar.gzip. Los usuarios de Linux y de Mac (Stuffit) no deberían tener problemas con ninguno de los dos; los usuarios de Windows pueden no tener una utilidad para el último.

Sólo Windows

Los usuarios de Windows pueden descargar el instalador binario para Windows OmegaT_1.4.4_setup.exe y ejecutarlo. En este caso, el instalador crea un acceso directo para ejecutar OmegaT en el menú Inicio > Programas.

Configuración (sólo Linux)

Si está utilizando Linux pero no se encuentra aún muy familiarizado con este sistema operativo, puede hacer que la ejecución de OmegaT sea más fácil de la siguiente forma:

Cree un guión de shell en un archivo de texto con el siguiente contenido:

#!/bin/bash
cd <OmegaT>
<java>/java -jar OmegaT.jar

en donde <OmegaT> es el directorio que contiene el archivo OmegaT.jar y <java> es el directorio que contiene el binario del J2RE (el archivo denominado java). Si ha configurado la ruta de acceso a Java, no necesita especificar este directorio, y java -jar OmegaT.jar es suficiente. Es necesario especificar el directorio si no se ha configurado la ruta de acceso a Java, y también si su sistema tiene más de una versión de Java instalada y desea asegurarse de que OmegaT sea ejecutado por la versión adecuada.

Guarde el archivo de texto en su escritorio como OmegaT, y cambie los permisos del mismo para hacerlo ejecutable (en GNOME y KDE, por ejemplo, menú contextual > Propiedades > Permisos > Ejecutar). Ahora puede lanzar OmegaT simplemente haciendo clic sobre el archivo OmegaT. Si quiere utilizar el icono del programa, lo encontrará en /OmegaT/images.

Ejecutar la aplicación

Macintosh OS X

Haga clic en el archivo OmegaT.jar en el administrador de archivos.

Windows

Si se ha instalado OmegaT con ayuda del instalador para Windows (ver más arriba), puede ejecutarse seleccionando el acceso directo a OmegaT 1.4.4 situado en el menú Inicio > Programas > OmegaT. Puede también hacer clic en los archivos OmegaT.exe, OmegaT.bat o OmegaT.jar en el administrador de archivos.

Linux

Si no ha realizado el proceso de configuración arriba indicado, ejecute java -jar OmegaT.jar desde la línea de mandatos. Observe la tipografía de OmegaT: las mayúsculas son importantes.

Si no ha configurado Java para detectar la ubicación de los archivos de OmegaT, debe en primer lugar desplazarse al directorio que contiene el archivo OmegaT.jar.

Si no configurado la ruta de acceso a Java, debe especificar la ruta completa. Si no ha hecho nada de lo anterior, su procedimiento de ejecución podría parecerse al siguiente:

cd /home/usuario/OmegaT
/usr/java/j2re1.4.2/bin/java -jar OmegaT
