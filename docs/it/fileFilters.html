Introduzione

OmegaT mette a disposizione una serie di filtri altamente personalizzabile che consente di impostare la maggior parte degli aspetti della loro operatività.

I filtri dei file sono parti di codice in grado di:

  • Leggere il documento in un particolare formato di file. Per esempio, file di solo testo.
  • Estrarre dal file il contenuto che dovrà essere tradotto.
  • Automatizzare le modifiche dei nomi dei file documento tradotti, sostituendo il contenuto traducibile con la traduzione effettuata.

Tipi di file

Quando OmegaT apre un file dalla propria directory sorgente, cercherà di selezionare il filtro servendosi dell'estensione associata al file stesso. Più precisamente, OmegaT cerca di associare i modelli dei nome dei file di partenza del filtro al nome del file stesso. Per esempio, il modello *.xhtml verrà associato ai file con estensione .xhtml. Se viene reperito il filtro appropriato, il file viene aperto ed è pronto per essere lavorato. Per esempio, per default, per poter lavorare sui file con estensione .xhtml verrà utilizzato il filtro XHTML. Si ha la possibilità di modificare o di aggiungere modelli di nomi di file che dovranno essere gestiti dai vari filtri dei file.

Codifica dei file

I file di solo testo e i file HTML possono essere memorizzati con diverse codifiche. I file HTML possono contenere meta-istruzioni di codifica che verranno utilizzate da OmegaT per leggere il file; i file di testo, al contrario, non sono associati ad alcun elemento per poterli definire. Per i file di solo testo e per i file HTML senza dichiarazione di codifica OmegaT utilizzerà la codifica impostata per una specifica estensione. Per esempio, per default, i file .txt verranno caricati utilizzando la codifica di default del Sistema Operativo (OS). Per ogni modello di nome dei file di partenza si ha la possibilità di modificare la codifica originaria.

I file HTML e di solo testo possono anche venire scritti utilizzando una qualsiasi codifica. Per esempio, se si traduce un testo dal francese al giapponese, sarebbe opportuno impostare un filtro che utilizzi una delle codifiche proprie della lingua giapponese di arrivo, oppure usare la codifica Unicode UTF-8, UTF-16 o UTF-32. Per default, la codifica salvata con il file è la medesima utilizzata quando si è caricato un file di solo testo, mentre per i file HTML viene adottata la codifica UTF-8. Si tenga presente che i file HTML aggiungono al file di arrivo una dichiarazione di codifica che consente al navigatore di poterlo leggere correttamente, in modo da non visualizzare caratteri strani.

Assegnazione di un nuovo nome ai file tradotti

A volte, potrebbe essere necessario assegnare automaticamente un nuovo nome al/ai file che vengono tradotti; è il caso, per esempio, dell'aggiunta di un codice di linga al nome del file finale. Questa operazione potrà essere anche eseguita a livello della finestra di dialogo "Filtri dei file", utilizzando un modello di nome di file appropriato.

Impostazione dei filtri dei file

La maggior parte degli utenti non dovrebbe avere problemi quando utilizza le opzioni predefinite dei vari filtri dei file. Se necessario, tuttavia, si potrà aprire la finestra principale selezionando il comando Opzioni -> Filtri dei file... dalla barra dei menu. Si tenga presente che è possibile modificare le opzioni dei filtri solo nel caso in cui non vi sia un progetto già aperto, in quanto tali modifiche avranno effetto sulla creazione dei file tradotti.

Finestra di dialogo "Filtri dei file"

La finestra contiene un elenco dei filtri dei file disponibili.

Se non si desidera utilizzare OmegaT per tradurre i file di alcuni tipi, sarà sufficiente disattivare la casella corrispondente al tipo di file, presente nella colonna "Abilitato". In questo caso, OmegaT, quando caricherà i progetti, ignorerà questo tipo di file e, quando creerà i documenti di arrivo, li copierà senza apportarvi alcuna modifica. In un momento successivo, quando si deciderà di utilizzare di nuovo questo filtro, sarà sufficiente attivarne di nuovo la casella di controllo corrispondente.

Se si sono eliminate tutte le configurazioni, fare clic sul pulsante Predefiniti: questa operazione riporta la configurazione al suo stato di lavoro predefinito.

Per modificare il modo in cui il filtro interviene sui file e sulla loro codifica, selezionare dall'elenco il filtro e fare clic su Modifica....

Finestra di dialogo "Modifica filtro"

Questa finestra di dialogo consente di impostare i modelli dei nomi dei file di partenza per quei file che dovranno essere elaborati dal filtro, di personalizzare i nomi dei file tradotti e di selezionare la codifica da adottare quando viene caricato il file di partenza e salvato quello di arrivo tradotto.

Modello del nome di file di partenza utilizzando le regole della maschera dei file DOS. L'uso del carattere * viene utilizzato per sostituire un qualsiasi numero di caratteri o simboli; il carattere ? sostituisce, invece, un singolo carattere/simbolo, lasciando inalterati tutti gli altri caratteri/simboli. Per esempio, se si desidera utilizzare il filtro di testo per la gestione dei file "Leggimi" (leggimi, leggi.mi, e leggimi.txt) si dovrebbe usare il modello leggi*.

I campi di codifica del file di partenza e di quello di arrivo utilizzano caselle combinate, contenenti tutte le codifiche riconosciute. Si tenga presente che non tutti i filtri consentono di selezionare una codifica, poiché possono lavorare su un formato di file associato a una sola codifica alla volta. Se, per i formati di file che consentono di scegliere la codifica, si imposta la codifica su "<auto>", ciò significa che si lascia a OmegaT il compito di decidere automaticamente la selezione. Il processo adottato è il seguente:

  • OmegaT cercherà di indovinare la codifica associata ai file HTML/XHTML.
  • OmegaT utilizzerà la codifica di default adottata dal sistema operativo quando deve gestire i file di solo testo.

Il modello del nome del file di arrivo utilizza una sintassi particolare; pertanto, se si desidera intervenire su questo campo, sarebbe opportuno fare clic su Modifica... per accedere alla finestra di dialogo "Modifica filtro".

Se si desidera che il filtro gestisca i file che corrispondono a uno o più modelli di nome di file di partenza, fare clic su Aggiungi... per accedere alla finestra di dialogo "Aggiungi modello".

Se si desidera ritornare alla configurazione predefinita del filtro, fare clic su Predefiniti.

Finestra di dialogo "Aggiungi/Modifica modello"

Sia che si desideri aggiungere un nuovo modello o modificarne uno esistente, verrà comunque proposta la stessa finestra di dialogo. L'unica differenza si trova a livello dell'editor speciale del modello del nome di file di arrivo, mentre tutti gli altri campi saranno quelli più sopra descritti.

La sezione del modello del nome di file di arrivo consente di comporre, in modi diversi, il nome del file di arrivo tradotto

  • Per default viene assunto il valore ${filename} -- ovvero il nome del file di partenza completo dell'estensione associata; in questo modo il nome del file tradotto sarà uguale a quello del file di partenza.
  • ${nameOnly} -- consente di inserire solo il nome del file di partenza senza associarlo all'estensione.
  • ${extension} -- definisce l'estensione del file di partenza.
  • ${targetLocale} -- codice locale di arrivo (nella forma "xx_YY").
  • ${targetLanguage} -- lingua e codice nazione di arrivo (nella forma "XX-YY").
  • ${targetLanguageCode} -- solo il codice della lingua di arrivo (nella forma "XX").
  • ${targetCoutryCode} -- solo il codice della nazione di arrivo (nella forma "YY").

Su richiesta, e se ci comunicherete la loro funzione, siamo in grado di aggiungere altre variabili. È sufficiente accedere a uno dei nostri gruppi di utenti.
