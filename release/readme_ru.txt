  2.  Что такое OmegaT?

OmegaT - это программа перевода с помощью компьютера (CAT, Computer Assisted Translation). Это свободная программа, то есть, вам не нужно платить за её использование (в том числе в профессиональных целях). Кроме того, вы свободны модифицировать и/или распространять OmegaT, соблюдая пользовательскую лицензию.

Главные преимущества OmegaT таковы:
  - запускается на любой операционной системе, поддерживающей Java
  - может использовать любые корректные TMX-файлы в качестве памяти переводов
  - гибкая сегментация на уровне предложения (используется методика, схожая с SRX)
  - поиск в проекте и во внешних файлах памяти переводов
  - поиск файлов поддерживаемых форматов в любом каталоге 
  - подбор нечётких совпадений
  - интеллектуальная обработка проектов,в том числе со сложными иерархиями каталогов
  - поддержка глоссариев (терминологическая справка)
  - лёгкая для понимания и исчерпывающая документация
  - переведена на большое количество языков.

OmegaT непосредственно поддерживает файлы следующих форматов:
  - неформатированный текст (plain text)
  - HTML и XHTML
  - HTML Help Compiler
  - OpenDocument / OpenOffice.org
  - пакеты Java resource (.properties)
  - INI-файлы (файлы с парами "ключ=значение" любой кодировки)
  - PO-файлы
  - формат файлов документации DocBook
  - файлы Microsoft OpenXML
  - одноязычные XLIFF-файлы OKAPI

OmegaT можно настроить на обработку и других форматов файлов.

OmegaT автоматически обработает даже самую сложную иерархию исходных каталогов, найдёт все поддерживаемые файлы и создаст каталог перевода с идентичной структурой, включая копии всех неподдерживаемых файлов.

Чтобы быстро ознакомиться с возможностями OmegaT, запустите программу и прочитайте Руководство по Быстрому Старту, которое отображается в главном окне.

Полное руководство пользователя можно найти в меню "Справка" после запуска OmegaT.

 3. Установка OmegaT

3.1 В целом. OmegaT требует, чтобы на вашей системе была установлена Java Runtime Environment версии 1.4 или выше. OmegaT в настоящее время поставляется вместе с JRE, чтобы избавить пользователей от необходимости выбирать, искать и устанавливать её. 

Для пользователей Windows и Linux: если вы уверены в том, что в вашей системе уже установлена подходящая версия JRE, вы можете установить версию OmegaT без JRE (это отмечено в названии версии - "Without_JRE"). 
Если вы не уверены, то мы рекомендуем использовать "стандартную версию", включающую JRE. Это безопасно, поскольку даже если JRE уже установлена в вашей системе, эта версия не будет с ней конфликтовать.

Пользователям Linux нужно обратить внимание на то, что OmegaT не будет работать с открытыми/свободными реализациями Java, которые можно найти во многих Linux-дистрибутивах (например, в Ubuntu), поскольку они либо устарели, либо неполны. Скачайте и установите Sun's Java Runtime Environment (JRE) по ссылке выше или скачайте и установите OmegaT в комплекте с JRE (архив tar.gz, помеченный "Linux").

У пользователей MacOSX JRE обычно уже установлена.

Если вы запускаете Linux на PowerPC, то вам придётся скачать IBM's JRE,
поскольку Sun не предоставляет JRE для PPC-систем. Это можно сделать по адресу:
    http://www-128.ibm.com/developerworks/java/jdk/linux/download.html 

3.2 Установка
Чтобы установить OmegaT, просто создайте подходящий каталог для неё (например, C:\Program 
Files\OmegaT в Windows или /usr/local/lib в Linux). Скопируйте zip-архив с OmegaT в этот каталог и распакуйте его там.

3.3 Запуск OmegaT

3.3.1 Для пользователей Windows

OmegaT можно запустить несколькими способами.

* Дважды щёлкните мышкой на  файле OmegaT-JRE.exe, если вы используете версию с встроенной JRE или на OmegaT.exe в ином случае.

* Дважды щёлкните на файле OmegaT.bat. Если в файловом менеджере (Проводнике Windows) вы видите только файл OmegaT, но не OmegaT.bat, измените настройки так, чтобы были видны расширения файлов.

* Дважды щелкните по файлу OmegaT.jar. Это сработает только если тип файлов .jar в вашей системе ассоциирован с Java.

* Из командной строки. Запустить OmegaT можно следующими командами:

  cd <каталог, где расположен файл OmegaT.jar>

  <имя и путь к исполняемому файлу Java> -jar OmegaT.jar

(Исполняемый файл Java - это файл java.exe.
Если Java установлена и настроена на системном уровне, полный путь к нему вводить не обязательно).

Вы можете перетащить файлы OmegaT-JRE.exe, OmegaT.exe или OmegaT.bat на рабочий стол или в стартовое меню, чтобы вызывать их оттуда.

3.3.2 Для пользователей Linux

* Из командной строки выполните:

  cd <каталог, где расположен файл OmegaT.jar>

  <имя и путь к исполняемому файлу Java> -jar OmegaT.jar

(Исполняемый файл Java - это файл java. Если Java установлена и настроена на системном уровне, полный путь к нему вводить не обязательно.)


3.3.2.1 Для пользователей Linux и KDE

Вы можете добавить OmegaT в меню следующим образом:

Центр Управления KDE - Рабочий стол - Панели - Меню - Редактировать меню К - Файл - Новый элемент/Новое подменю.

Затем, после выбора подходящего меню, добавьте новый элемент или подменю при помощи Файл - Новое подменю или Файл - Новый элемент. Введите OmegaT в качестве имени нового элемента.

В поле "Команда" найдите скрипт, запускающий OmegaT, и  выберите его. 

Щелкните на кнопке выбора иконки (справа от полей Имя/Описание/Комментарий) - Прочие значки - Обзор и зайдите в подкаталог /image в каталоге OmegaT. Выберите иконку OmegaT.png.

Сохраните изменения при помощи Файл - Сохранить.

3.3.2.2 Для пользователей Linux и GNOME.

Вы можете добавить OmegaT в вашу панель (меню вверху экрана) следующим образом:

Щелкните правой кнопкой мыши на панели - Add New Launcher. Введите "OmegaT" в поле "Name"; используйте кнопку обзора, чтобы ввести путь к запускающему OmegaT скрипту в поле "Command". Выберите скрипт и подтвердите выбор нажатием кнопки OK.

 5.  OmegaT работает неправильно? Вам нужна помощь?

Прежде, чем сообщать об ошибке, убедитесь, что вы внимательно прочли документацию. Возможно, это не ошибка, а особенность OmegaT. Если в лог-файле OmegaT вы видите слова вроде "Error", "Warning", "Exception" или "died unexpectedly", тогда, вероятно, вы действительно обнаружили проблему. Файл log.txt расположен в каталоге пользовательских настроек, подробнее смотри руководство.

Ваш следующий шаг - удостовериться, что другие пользователи тоже наблюдают эту ошибку и что никто не сообщил о ней до вас. Для этого просмотрите страницу сообщений об ошибках на SourceForge. Вы должны размещать сообщение об ошибке только если вы уверены, что вы первый, кто нашёл некую воспроизводимую последовательность действий, которая вызывает нечто неправильное.

Хорошее сообщение об ошибке включает три компонента:
  - Как воспроизвести,
  - Что вы ожидали увидеть, и 
  - Что увидели вместо этого.

Вы можете прикреплять копии файлов, сегменты логов, скриншоты - всё, что по вашему мнению, поможет разработчикам найти и исправить ошибку.

Чтобы просмотреть архивы списка рассылки пользователей OmegaT, зайдите на
     http://groups.yahoo.com/group/OmegaT/

Чтобы просмотреть страницу сообщений об ошибках и отправить новое сообщение, зайдите на
     http://sourceforge.net/tracker/?group_id=68187&atid=520347

Чтобы отслеживать состояние вашего сообщения об ошибке, вы, возможно, захотите зарегистрироваться на SourceFourge.
